\# 项目简历描述精修版：端侧 AI 模型部署与推理优化



\## 项目名称



基于 PyTorch 与 ONNX Runtime 的轻量化图像分类模型部署与推理优化系统



\## 技术栈



Python、PyTorch、torchvision、ONNX、ONNX Runtime、NumPy、Pandas、Matplotlib、Pillow、Git



\## 简历 Bullet 版本一：稳妥版



\* 基于 ImageNet 预训练 ResNet18 构建图像分类推理流程，完成真实图片读取、图像预处理、Top-K 分类输出和推理延迟统计。

\* 使用 torch.onnx.export 将 PyTorch 模型导出为 ONNX 格式，并通过 onnx.checker.check\_model 验证模型结构合法性。

\* 基于 ONNX Runtime 实现 ONNX 模型 CPU 推理，对比 PyTorch 与 ONNX Runtime 的 Top-K 分类结果、logits 数值误差、平均延迟和 FPS。

\* 设计 batch size = 1、2、4、8 的推理性能测试流程，生成 CSV 结果与性能曲线；实验中 ONNX Runtime 在 CPU 上相比 PyTorch 实现约 2.00x 到 2.67x 推理加速，输出误差稳定在 1e-5 量级。



\## 简历 Bullet 版本二：更工程化版



\* 搭建 PyTorch 到 ONNX Runtime 的端侧 AI 推理部署流程，覆盖模型加载、真实图片预处理、ONNX 导出、推理验证、性能测试与可视化分析。

\* 使用 ImageNet 预训练 ResNet18 作为基线模型，完成 PyTorch 单图推理与 Top-K 分类结果输出，并统计 CPU 推理延迟与吞吐率。

\* 将 PyTorch 模型导出为 ONNX 格式，基于 ONNX Runtime 完成 CPU 部署推理，并通过 Top-K 一致性与 logits 最大绝对误差验证模型转换正确性。

\* 构建 batch size 性能 benchmark，测试 batch size = 1、2、4、8 下的 batch latency、image latency、FPS 与 speedup；实验结果显示 ONNX Runtime 相比 PyTorch 获得约 2 倍以上 CPU 推理加速。



\## 简历 Bullet 版本三：冲端侧 AI / 模型部署岗位版



\* 围绕端侧 AI 模型部署场景，构建 PyTorch → ONNX → ONNX Runtime 的轻量化图像分类推理优化系统，实现模型转换、部署推理、性能 benchmark 与结果可视化。

\* 基于 ResNet18 完成真实图片分类推理，使用 torchvision 官方预处理流程保证输入一致性，并输出 Top-K 分类结果、平均推理延迟和 FPS。

\* 通过 ONNX Runtime 部署 ONNX 模型，对比 PyTorch 与 ONNX Runtime 的分类结果和 logits 数值误差，验证模型转换后输出一致性，最大误差约为 1e-5。

\* 设计不同 batch size 下的 CPU 推理性能实验，生成 CSV 和可视化曲线；实验中 ONNX Runtime 在 batch size = 1、2、4、8 下均保持明显性能优势，最高实现约 2.67x 推理加速。
