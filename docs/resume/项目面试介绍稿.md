\# 项目面试介绍稿：端侧 AI / ONNX Runtime 部署项目



\## 1 分钟介绍版



我做了一个基于 PyTorch 与 ONNX Runtime 的端侧 AI 推理部署项目，主要目标是把训练框架里的模型转成更适合部署的 ONNX 格式，并验证 ONNX Runtime 在 CPU 推理场景下的加速效果。



项目里我使用 ImageNet 预训练 ResNet18 作为基础模型，先完成 PyTorch 真实图片分类推理，然后用 torch.onnx.export 导出 ONNX 模型，再用 ONNX Runtime 加载模型进行 CPU 推理。为了验证模型转换是否正确，我对比了 PyTorch 和 ONNX Runtime 的 Top-K 分类结果以及 logits 最大误差，结果显示 Top-K 一致，误差约为 1e-5。



在性能测试部分，我设计了 batch size = 1、2、4、8 的 benchmark，统计 batch latency、单图延迟、FPS 和 speedup，并用 Matplotlib 生成性能曲线。实验结果显示，ONNX Runtime 相比 PyTorch 在 CPU 上实现了约 2 倍以上的推理加速。这个项目让我完整跑通了从 PyTorch 模型到 ONNX Runtime 部署推理的基本流程。



\## 3 分钟介绍版



这个项目的背景是端侧 AI 和边缘智能场景中，模型不能只停留在训练框架里，还需要转换成适合部署和推理优化的格式。因此我做了一个基于 PyTorch 与 ONNX Runtime 的轻量化图像分类模型部署与推理优化系统。



项目整体分为四个部分。



第一部分是 PyTorch 推理流程。我使用 torchvision 中的 ImageNet 预训练 ResNet18 作为基础模型，完成真实图片读取、图像预处理、batch 维度构造、模型前向推理、Top-K 分类结果输出和推理延迟统计。这里主要是建立一个可靠的 PyTorch baseline。



第二部分是 ONNX 模型导出。我使用 torch.onnx.export 将 PyTorch 模型导出为 ONNX 格式，并使用 onnx.checker.check\_model 检查模型格式是否合法。导出时我也处理了 opset version 和动态 batch size 的问题，后续 batch benchmark 使用的是支持动态 batch 的 ONNX 模型。



第三部分是 ONNX Runtime 推理验证。我使用 ONNX Runtime 创建 inference session，读取模型的 input name 和 output name，然后把 PyTorch 预处理后的 tensor 转成 NumPy 输入给 ONNX Runtime。为了验证导出后的模型是否正确，我对比了 PyTorch 和 ONNX Runtime 的 Top-K 分类结果以及 logits 最大绝对误差。实验结果显示，两者分类结果一致，最大误差约为 1e-5，说明 ONNX 模型基本保持了 PyTorch 模型的输出一致性。



第四部分是性能 benchmark。我设计了 batch size = 1、2、4、8 的性能测试，分别统计 PyTorch 和 ONNX Runtime 的 batch latency、image latency、FPS 和 speedup，并把结果保存成 CSV，再用 Matplotlib 画出性能曲线。实验结果显示，在 CPU 推理场景下，ONNX Runtime 相比 PyTorch 在不同 batch size 下都有明显加速，整体约为 2.00x 到 2.67x。



这个项目的价值在于，它不是只做模型训练，而是完整覆盖了模型部署链路中的模型转换、推理验证、性能测试和结果可视化。后续我还计划继续扩展 MobileNetV2 对比、ONNX 动态量化、内存占用测试，以及在 Linux 或真实边缘设备上的部署测试。



\## 面试官可能追问 1：为什么 ONNX Runtime 比 PyTorch 快？



PyTorch 更偏训练框架，虽然也可以推理，但它的动态图机制和通用性设计会带来一定额外开销。ONNX Runtime 是专门面向推理优化的运行时，可以对计算图做优化，比如算子融合、图优化、内存规划和执行调度优化。因此在 CPU 推理场景下，ONNX Runtime 往往能比直接使用 PyTorch 推理更快。



\## 面试官可能追问 2：怎么证明 ONNX 模型没有导错？



我主要从两个角度验证。第一是结构层面，使用 onnx.checker.check\_model 检查导出的 ONNX 文件是否合法。第二是输出层面，对同一张图片分别用 PyTorch 和 ONNX Runtime 推理，比较 Top-K 分类结果和 logits 最大绝对误差。我的实验中 Top-K 结果一致，最大误差约为 1e-5，因此可以认为导出结果是正确的。



\## 面试官可能追问 3：batch size 增大为什么 FPS 会提升？



batch size 增大后，模型一次前向可以处理多张图片，虽然单次 batch latency 会增加，但平均到每张图片上的 image latency 会下降，CPU 的计算资源也能被更充分利用，所以整体 FPS 会提升。这就是批量推理提高吞吐率的原因。



\## 面试官可能追问 4：这个项目和端侧 AI 有什么关系？



端侧 AI 的核心问题之一是如何让模型在资源受限环境中高效运行。这个项目虽然是在 PC 的 CPU 环境上完成，但流程本身对应端侧部署的基本链路：训练框架模型转换、推理引擎部署、延迟和吞吐测试、结果一致性验证。后续可以进一步迁移到边缘设备或使用 TensorRT、OpenVINO、NCNN 等更贴近端侧硬件的推理后端。
